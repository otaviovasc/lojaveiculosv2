Codex Session Delivery
1) Public Storefront Lead CapturePublic listing pages now collect buyer interest, validate the payload, create a store-scoped CRM lead, emit audit context, and expose the contract in OpenAPI and AI-facing docs. Reviewer hardening added per-client throttling and duplicate-lead suppression before CRM writes.
2) Documents WorkspaceAuthenticated store actors can list documents across vehicle, lead, sale, payment, finance, and fiscal contexts through a permissioned backend service and responsive frontend workspace. The workspace now opens a document detail panel with preview, download, regeneration, void, status, and operation history actions.
3) Editable Document TemplatesStores can customize reservation receipts, sale contracts, sale receipts, delivery terms, and power-of-attorney templates with clause editing, variable preview, default restore, template persistence, audited generation metadata, server-side variable interpolation, customized generated titles, and regenerate flows that preserve operation history.
4) Exact Permission AdministrationAgencies can manage store owners, salespeople, supervisors, and investors with role assignability rules; owners can manage non-privileged store users; the UI now exposes inherited, allowed, and denied permission states explicitly. Invalid or duplicate permission overrides are rejected instead of silently dropped.
5) Private Document Hosting + VersionsGenerated workflow documents now write to the object storage port, use Cloudflare R2 signed GET descriptors for downloads, avoid durable public document URLs, append immutable version rows on regeneration, and expose newest-first version history in the operator document panel. The mutable document row is the current pointer; document_versions is append-only history.
6) R2 Media PolishInventory media now shows public readiness, public photo count, hidden media count, and a direct cover action that publishes and reorders the selected photo. Public storefront cards render the first public photo by display order, and listing details expose an ordered gallery selector.
7) Admin Observability DashboardInternal health now computes critical/warning/healthy status from audit events and open sink failures. The admin dashboard exposes alert cards, summary KPIs, outcome/severity charts, top actions, actor activity, sink impact, and recent audit rows for the current tenant/store scope.
8) Agency Billing + Marketplace FoundationBilling now exposes agency store allocations, financial KPIs, entitlement matrix rows, reasoned entitlement changes, and a scoped history feed. Marketplaces now have entitlement and exact permission keys plus OLX/Mercado Livre account and sync-job APIs, a mounted frontend console, OpenAPI docs, and AI-facing documentation.
9) Real Marketplace Provider RuntimeProvider accounts now use OAuth connection URLs and authenticated completion, credentials are encrypted at rest and redacted from responses, listing sync jobs execute through provider gateways, scoped listing payloads are mapped for Mercado Livre/OLX, and provider listing ids are persisted for future updates/unpublish operations.
Foundation Guardrails (Now)
Development and product decisions locked for Store, Public Site, Roles, and Billing delivery.
Store profile schemaMandatory fields (legal name, slug, timezone, currency, contact block, publish visibility) are enforced and validated.
Public storefront read pathNow resolves only published public site settings to prevent draft leakage to production traffic.
Transactional repository writesStore settings repository writes are now fully transactional for consistency and rollback safety.
Unified permission read guardUnified settings read requires both store_profile.manage and store_public_site.manage.
Domain validationAmbiguous hostnames are now rejected in domain validation before persistence.
Quality gatesCustom domain + SSL contract, SEO defaults, R2 media constraints, and WhatsApp/contact routing were added under test coverage.
Test coverageGuardrails and happy-path settings flows have automated tests and are now validate-green in CI.
Entitlements before permissionsBilling controls whether a feature exists for the store; roles control what each user can do inside enabled features.
Canonical vehicle workflowsReserved/sold states are blocked from generic edits and must go through reserve/sell workflows with documents and finance side effects.
Private document deliveryDocument downloads are service-authorized, scoped by tenant and store, and returned as short-lived signed URLs. Raw storage keys are not exposed in workspace DTOs, OpenAPI schemas, or the web document UI.
Public contract hardeningPublic storefront vehicle payloads expose slugs only; internal listing/media IDs stay server-side and UUID listing fallback is removed.
Finance link ownershipFinancial entries can only link to targets that exist inside the same tenant and store scope.
Policy-aware auditService contexts now route audit events through failure policy handling, so critical audit writes fail closed instead of being swallowed.
Scoped external APIAPI clients use hashed one-time keys, store scope, and least-privilege permission scopes; billing, users, audit, tenant, and settings remain user-only.
Integration safetyExternal API traffic is rate-limited from durable request logs, mutations require Idempotency-Key, and each request persists client/store/status/latency metadata.
Traceable logsShared service log metadata now includes actorId, actorKind, actorExternalId, requestId, storeId, tenantId, and correlationId where available.
Internal health dashboardThe Saude module reads scoped audit events and open audit-sink failures from the separated audit database.
